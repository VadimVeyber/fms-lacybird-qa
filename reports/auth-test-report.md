- Переход на вкладку «Email» — работает

- После отправки — появляется поле для 6-значного кода
- **Скриншоты:** `01` → `02` → `03` → `04`

### Вход по сохранённой сессии

- **Скриншот:** `05-auth-logged-in-erp.png`

## 2. Негативные сценарии

- **Поведение корректное** ✅
- **Скриншоты:** `09`, `10`

## 4. 🔴 Уязвимость: подстраницы ERP доступны без авторизации

### Описание
Корневая страница `/erp` корректно редиректит неавторизованных пользователей на логин.
Однако **прямой переход по URL на любую подстраницу** открывает её без проверки сессии:

| Страница | Без авторизации |
|----------|----------------|
| `/erp` | ✅ Редирект на логин |
| `/erp/materials` | 🔴 Открывается |
| `/erp/receipts` | 🔴 Открывается |
| `/erp/settings` | 🔴 Открывается |
| `/erp/productions` | 🔴 Открывается |

### Что видит неавторизованный пользователь
Полный доступ к данным: каталоги материалов, приёмки, настройки (включая управление пользователями), производственные документы.

### Скриншоты
- `SECURITY-erp_materials.png` — каталог материалов без логина
- `SECURITY-erp_settings.png` — настройки без логина
- `SECURITY-erp_receipts.png` — приёмки без логина
- `SECURITY-erp_productions.png` — производство без логина

### Рекомендация
Добавить серверную проверку сессии на **все** маршруты `/erp/*`, не только на корень. В Chatium-фреймворке это, скорее всего, middleware или guard в `index.tsx` модуля ERP.
